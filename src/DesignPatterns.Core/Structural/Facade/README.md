# Patrón Facade

## ¿Qué es Facade?

Facade proporciona una interfaz unificada y simplificada a un conjunto de interfaces en un subsistema. Oculta la complejidad del sistema.

## Propósito

- Simplificar interfaz compleja
- Desacoplar cliente de subsistema
- Facilitar uso de componentes complejos
- Reducir dependencias

## Ventajas ✅

- ✅ Simplifica código cliente
- ✅ Desacopla de subsistema
- ✅ Reduce complejidad
- ✅ Punto único de entrada

## Desventajas ⚠️

- ⚠️ Facade puede crecer demasiado
- ⚠️ Puede ocultar funcionalidad útil
- ⚠️ Abstracción no siempre clara
- ⚠️ Difícil mantener balance

## Cuándo usar

✓ Sistemas complejos con múltiples clases
✓ Simplificar interfaz para clientes
✓ Desacoplar cliente de implementación
✓ Capas de aplicación

## Cuándo NO usar

✗ Sistemas simples
✗ Sin subsistemas complejos
✗ Cuando claridad es más importante
